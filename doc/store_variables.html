内置变量¶

Ren'Py 拥有许多控制其函数运行的内置变量。内置变量随时可以改变。游戏开始后发生改变的内置变量将可以通过存档系统保存和读取并通过回滚系统复原。

adv = Character(...)¶

设置文字冒险模式下的角色模板，这个角色模板将作为 Character() 函数执行时的默认值使用。

_game_menu_screen = "save"¶

这个视屏将在没有选择特殊视屏的情况下进入游戏菜单时显示。（例如右键、按下 esc 键或者执行 ShowMenu() 函数没有传递参数时。）若设置为 None, 则禁止打开游戏菜单。

在开始画面中这个变量被设置为了 None，离开开始画面后则恢复成了默认值。

main_menu = False¶

在主菜单中 Ren'Py 将这个变量的值设置为了 True 。可以使用这个变量来让视屏在处于主菜单时发生改变。

menu = renpy.display_menu¶

设置在游戏中打开菜单时执行的函数。这个函数的参数应该与 :func`renpy.display_menu` 相同。将这里设置为 nvl_menu() 将打开视觉小说模式的菜单。

mouse_visible = True¶

设置鼠标是否可见。当进入游戏的标准菜单时这个变量将自动设置为真。

name_only = Character(...)¶

在角色名称为字符串的情况下的角色模板。代码：

"Eileen" "Hello, world."

等同于：

$ temp_char = Character("Eileen", kind=name_only)
temp_char "Hello, world."

除非 temp_char 变量已经使用过了。

narrator = Character(...)¶

（Say命令没有设置角色或者角色姓名的时候）旁白的角色模板。代码：

"Hello, world."

等同于：

narrator "Hello, world."
_rollback = True¶

设置是否允许回滚模式。

say = ...¶

Ren'Py 用于显示对话时执行的函数。调用这个函数时传递了三个参数。第一个参数 (who) 为说出这句对话的角色（若为旁白则为 None）。第二个参数(what)为对话的内容。

第三个参数是一个名为 interact 的关键字参数，默认为 True。当为真的情况下对话命令将等待玩家的按键，若为假则将立刻返回。

由于可以很简单的使一个角色做出对话，通常情况下很少直接调用这个函数。这个变量更主要是为了给对话命令重新定义钩子方法而存在的。

save_name = ""¶

包含在存档内的存档名称。

_window = False¶

这个变量由 window show 和 window hide 命令控制，并间接的受到 window auto 的控制。若设置为真，即使没有对话命令也将立刻显示对话窗口。

_window_auto = False¶

这个变量将被 window auto 设置为真，被 window show 和 window hide 设置为假。如果设置为真则将开启窗口自动管理。

_window_subtitle = ''¶

这个变量的值将跟在 config.window_title 之后从而组成完整的游戏标题。在菜单中，这个变量的值将被自动设置为 config.menu_window_subtitle 。
