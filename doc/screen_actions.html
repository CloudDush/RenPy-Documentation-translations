视屏操作，值和函数¶

Ren'Py有一些专为视屏语言配套设计的操作，值以及函数。

操作¶

视屏操作会在按钮被激活，获焦或者失焦时被执行。某些操作会决定按钮是否处于敏感和选中状态。

除了这些操作外，action属性也可以是一个不带任何参数的函数。当需要执行操作时，这个函数将被执行。如果这个函数有返回值，它会作为交互的返回值被返回。

action也可以是一个由操作组成的列表，这些操作会被依次执行。

控制操作¶

以下这些操作可以管理视屏，处理交互结果，以及处理控制流。

Hide(screen, transition=None)¶

该操作将会使名为screen的视屏被隐藏，如果它处于显示状态的话。

transition
如果不是None，则为隐藏该视屏时使用的转场特效。
Jump(label)¶

将导致脚本执行器跳转到指定的标签。这个操作可以和renpy.run_screen配合使用，来定义一个用于决定目的地的热点地图。

NullAction()¶

什么也不做。

这个操作可以使按钮能够对hover和unhover事件作出响应，但是点击它不会发生任何事情。

Return(value=None)¶

该操作将返回给定的值，作为交互的返回值。这个操作经常用于菜单和热点地图，这样系统便能知道玩家做出的选择到底是什么。

在菜单中的话，该操作将退出菜单。

Show(screen, transition=None, *args, **kwargs)¶

此操作会显示另一个视屏。screen是该视屏的名称。关键字参数会传递给被显示的视屏。

如果不是None，系统将使用transition作为转场特效。

ShowTransient(screen, *args, **kwargs)¶

显示一个过渡视屏。该视屏将在此交互结束后被隐藏。

数据操作¶

这些操作可以设置或者调整数据。

SetDict(dict, key, value)¶

此操作将会把dict字典里的键key的值设置为value。

SetField(object, field, value)¶

将一个对象的域设置为一个给定值。 object是要设置的对象，field是域名的字符串，value是要设置成的值。

SetScreenVariable(name, value)¶

将导致当前视屏作用域内的变量name的值变成value。

SetVariable(variable, value)¶

将变量variable的值设置为value。

ToggleDict(dict, key, true_value=None, false_value=None)¶

切换dict字典的key的值。切换的意思是如果原来它的值是True（或者给定的值），那么将改成False（或者给定的值）；如果原来是False则改成True。其他情况下，不发生变化。

true_value
如果不是None，则为处于True状态下我们使用的值。
false_value
如果不是None，则为处于False状态下我们使用的值。
ToggleField(object, field, true_value=None, false_value=None)¶

切换object对象的field域。切换的意思和上面一样。

true_value
如果不是None，则为处于True状态下我们使用的值。
false_value
如果不是None，则为处于False状态下我们使用的值。
ToggleScreenVariable(name, true_value=None, false_value=None)¶

切换此视屏作用域内的变量name的值。

true_value
如果不是None，则为处于True状态下我们使用的值。
false_value
如果不是None，则为处于False状态下我们使用的值。
ToggleVariable(variable, true_value=None, false_value=None)¶

切换variable的值。

true_value
如果不是None，则为处于True状态下我们使用的值。
false_value
如果不是None，则为处于False状态下我们使用的值。

菜单操作¶

这些操作可以唤出菜单，或者主要是菜单内使用的动作。

MainMenu(confirm=True)¶

将导致Ren'Py返回主菜单。

confirm
若为True,Ren'Py会在退回主菜单时进行确认，而不是直接返回。
Quit(confirm=True)¶

退出游戏。

confirm
若为True，将先询问用户是否真的想退出游戏，而不是直接退出。
ShowMenu(screen=None, *args, **kwargs)¶

将导致玩家进入菜单。如果已经在菜单内里了，这将显示一个视屏或者跳到一个标签。

screen通常是视屏的名称。如果该视屏不存在，那么将在此名称后面加上 "_screen"，并跳转到该标签。

  • ShowMenu("load")
  • ShowMenu("save")
  • ShowMenu("preferences")

此操作也可以显示开发者定义的菜单视屏。比如说，如果定义了一个"stats"视屏，您可以将它作为菜单的一部分来使用，代码如下：

  • ShowMenu("stats")

不带参数的ShowMenu操作会进入默认的游戏菜单。该菜单的名称是_game_menu_screen决定的。您可以在游戏运行中修改它来做到在游戏的不同阶段进入不同的菜单。

额外的参数和关键字参数会传递给显示的视屏。

Start(label='start')¶

将导致Ren'Py的脚本执行器跳出菜单的环境，跳到给定的表情。主要用于从主菜单开始新游戏。一般的用法有：

  • Start() - 从start标签开始游戏。
  • Start("foo") - 从"foo"标签开始游戏。

文件操作¶

这些操作处理了存档，读档和删除存档文件相关的动作。有很多接受以下两个参数：

name
存档的名称。可以是一个字符串也可以是整数。它和页码page变量一起构成了存档文件名。
page
该操作使用的页码数。这个值是"auto", "quick",或者一个整数。如果是None，那么这个页码会被自动决定。
FileAction(name, page=None)¶

对文件做"正确的事情"。也就是说如果load菜单显示的话，就读档，否则就执行存档动作。

name
存档槽的名称。如果为None，那么将使用一个未被使用的存档槽。
page
存档或读档的页数。如果为None，将使用当前的页码。
FileDelete(name, confirm=True, page=None)¶

删除存档文件。

confirm
若为True，将向用户确认是否要删除档案文件。
FileLoad(name, confirm=True, page=None, newest=True)¶

读取存档。

name
读取存档的名称。如果不提供则无法读档。
confirm
若为True，将向用户询问是否要从该文件读取。
page
读取存档的页码。如果是None，则使用当前的页码。
newest
如果是true且按钮对应的存档是最新的存档，则按钮会处于选中状态。
FilePage(page)¶

将页码设置为page，这个参数的值应为"auto", "quick",或者一个整数。

FilePageNext(max=None, wrap=False)¶

前进到下一页。

max
如果设定该参数，那么应该是指定最大页数的数字。
wrap
如果为True，在最后一页使用该操作会回到第一页。（在设置了max的情况下）
FilePagePrevious(max=None, wrap=False)¶

返回前一页。（如果可能的话）

max
如果设定该参数，那么应该是指定最大页数的数字。（您必须设置该参数来启用下面的wrap参数）
wrap
若为True，在第一页使用该操作会回到最后一页。（在设置了max的情况下）
FileSave(name, confirm=True, newest=True, page=None, cycle=False)¶

保存档案。

如果按钮对应的存档是最新的存档，则按钮会处于选中状态。

name
存档槽的名称。如果是None则会使用一个未使用的存档槽。
confirm
如果为True，在覆盖原有存档时会提醒玩家即将覆盖档案。
newest
请无视此属性。（该功能已默认开启）
page
存档所在的页数。如果是None则使用当前的存档页数。
cycle
如果为True，那么存档将作为提供的页码的一号存档保存，而剩下的存档都会自动向后移动一位。
FileTakeScreenshot()¶

进行游戏保存时的截图。在存档视屏打开前调用该操作可以用来保证存档使用的截图是准确的。

QuickLoad()¶

进行快速读档。

QuickSave(message='Quick save complete.', newest=False)¶

进行快速存档。

message
当快速存档结束时显示的信息。
newest
如果此参数为True，那么快速存档会被标记为最新存档。

音频操作¶

Play(channel, file, **kwargs)¶

在指定频道播放音频文件。

channel
用于播放音频的频道。
file
用于播放的文件。

其他的关键字参数将传递给renpy.music.play()。

Queue(channel, file, **kwargs)¶

将音频文件加入指定频道的队列。

channel
用于播放音频的频道。
file
要播放的文件。

其他的关键字参数将传递给renpy.music.queue()。

SetMixer(mixer, volume)¶

设置mixer的音量为value.

mixer
要改变音量的mixer名称。是一个字符串，通常是 "music", "sfx", or "voice"之一。
value
目标音量。应在0.0和1.0之间（包括0.0和1.0）。
Stop(channel, **kwargs)¶

停止一个频道正在播放的音频。

channel
需要停止播放的频道。

其他关键字参数将传递给renpy.music.stop()。

其他操作¶

以下是剩下的操作。

Help(help=None)¶

显示帮助文件。

help
如果是游戏里的一个标签，那么将呼叫该标签。否则它应该是一个能在web浏览器里打开的文件。如果是None，那么将使用config.help，处理方法相同。
If(expression, true=None, false=None)¶

它返回true如果expression为true，否则返回false。您可以使用该操作来根据一个表达式的不同值做出不同的操作。注意如果使用None的话会让该按钮处于非敏感状态。

InvertSelected(action)¶

该方法将撤销指定操作的效果。

MouseMove(x, y, duration=0)¶

让鼠标指向x, y。如果设备没有鼠标或者_preferences.moust_move是False，那么什么也不会发生。

duration
该移动持续的时间，单位是秒。在这段时间里，鼠标会处于无法响应的状态。
Notify(message)¶

用renpy.notify()向用户显示信息message 。

OpenURL(url)¶

用网页浏览器打开给定的网址 url。

RollForward()¶

如果可以向前滚动的话，该操作将开始向前滚动剧情，否则该按钮处于非敏感状态。（正常剧情发展方向是向前发展）

Rollback()¶

如果可以向后滚动的话，该操作将开始向后滚动剧情。否则什么也不发生。

Screenshot()¶

截图。

SelectedIf(expression)¶

该操作使用一个表达式来决定一个按钮是否处于被选中状态。它应该作为一个或者更多操作组成的列表的一部分使用。比如：

#如果mars_flag为True，该按钮处于被选中状态。
textbutton "Marsopolis":
    action [ Jump("mars"), SelectedIf(mars_flag) ]
Skip(fast=False, confirm=False)¶

此操作会使游戏开始快进。如果在菜单中，将会返回游戏。

fast
如果为True，会直接跳到下一个选项处。
confirm
如果为True，在开始快进前向用户确认。
With(transition)¶

显示transition转场效果。

值¶

值是配合滑条使用的，用于绘制滑条，以及允许它们控制指定的值。

AnimatedValue(value=0.0, range=1.0, delay=1.0, old_value=None)¶

该操作动态地改变一个值，过程时长delay秒，值从 old_value变化到value。

value
目标的数值。
range
该值的范围，一个数字。
delay
变化时间。默认为1.0秒。
old_value
初始值。如果为None，那么我们将从被取代的AnimatedValue里面获取，如果有的话，否则它被初始化为value。
FieldValue(object, field, range, max_is_zero=False, style='bar', offset=0, step=None)¶

这个值允许您来调整一个对象上的一个域所对应的值。

object
要调整的对象。
field
要调整的域，一个字符串。
range
调整的范围。
max_is_zero

若为True，那么当域的值是0时，该滑条的值会变成range，然后所有的其他的值会变成1-当前值。（也就是满的时候值是0）

This is used internally, for some preferences.

style
滑条的样式。
offset
值的偏移量、
step
每次改变的最小值，默认情况下是滑条的1/10。
MixerValue(mixer)¶

音频mixer的音量对应的值。

mixer
要调整的mixer的名称，通常是 "music", "sfx",或 "voice"，但您可以自己创建新的mixer。
StaticValue(value=0.0, range=1.0)¶

该操作可以静态地创建一个值。

value
要创建的值，一个数字。
range
该值的范围。
VariableValue(variable, range, max_is_zero=False, style='bar', offset=0, step=None)¶
variable
字符串，提供需要调整的变量名称。
range
能够调整的范围。
max_is_zero

若为True，那么当变量的值是0时，该滑条的值会变成range，然后所有的其他的值会变成1-当前值。（也就是满的时候值是0）

对于有些设置，这个属性也被内部使用。

style
创建的滑条使用的样式。
offset
值的偏移量。
step
每次改变的最小值，默认情况下是滑条的1/10。
XScrollValue(viewport)¶

这个值是一个当前视屏上给定id的视端的x轴的值。视端必须在使用这个值的滑条之前定义。

YScrollValue(viewport)¶

这个值是一个当前视屏上给定id的视端的y轴的值。视端必须在使用这个值的滑条之前定义。

函数和类¶

这些函数和类在编写视屏时有用。

Preferences¶

虽然所有的设置都可以通过以上的操作和值来修改，但这需要对Ren'Py的一定认识才能做到。为了简化这一点，我们提供了Preferences这个方便的函数。您只需要提供合适的字符串作为参数，Preferences就会为您关联相应的设置选项。

Preference(name, value=None)¶

该函数会根据参数返回合适的值或操作。value应该是选项的名称， "toggle"，一个具体的值，或者留空（如果是按钮的话）。

能够作为按钮和hotspot的操作有：

  • Preference("display", "fullscreen") - 以全屏模式显示。
  • Preference("display", "window") - 以1x窗口模式显示。
  • Preference("display", 2.0) - 以2x窗口模式显示。
  • Preference("display", "any window") - 以之前的窗口模式显示。
  • Preference("display", "toggle") - 切换显示模式。
  • Preference("transitions", "all") - 显示所有转场特效。
  • Preference("transitions", "none") - 不显示转场特效。
  • Preference("transitions", "toggle") - 切换转场效果设置。
  • Preference("show empty window", "show") - 允许"window show"和"window auto"命令来显示空对话窗口。
  • Preference("show empty window", "hide") - 禁止以上命令执行。
  • Preference("show empty window", "toggle") - 切换以上的设置状态。
  • Preference("text speed", 0) - 文本立即显示。
  • Preference("text speed", 142) - 将文本速度设置为142个字符每秒。
  • Preference("joystick") - 显示摇杆设置。
  • Preference("skip", "seen") - 仅跳过已读文本。
  • Preference("skip", "all") - 跳过未读文本。
  • Preference("skip", "toggle") - 在以上两种模式间切换。
  • Preference("begin skipping") - 开始快进。
  • Preference("after choices", "skip") - 在选项后继续快进。
  • Preference("after choices", "stop") - 在选项后停止快进。
  • Preference("after choices", "toggle") - 在以上两种模式间切换。
  • Preference("auto-forward time", 0) - 将自动阅读的速度设置为无穷。
  • Preference("auto-forward time", 10) - 将自动阅读的速度设置为10（单位为250字/每单位时间）。
  • Preference("auto-forward", "enable") - 开启自动模式。
  • Preference("auto-forward", "disable") - 关闭自动模式。
  • Preference("auto-forward", "toggle") - 切换自动模式。
  • Preference("auto-forward after click", "enable") - 在单击后仍保留自动模式。
  • Preference("auto-forward after click", "disable") - 再单击后退出自动模式。
  • Preference("auto-forward after click", "toggle") - 在以上两种模式之间切换。
  • Preference("automatic move", "enable") - 开启自动鼠标模式。
  • Preference("automatic move", "disable") - 关闭自动鼠标模式。（与上面的模式的区别在于，之前是按照字数来阅读，而这个是在文本显示完毕后自动模拟点击一下鼠标）
  • Preference("automatic move", "toggle") - 切换自动鼠标模式。
  • Preference("wait for voice", "enable") - 自动模式中等待当前播放的语音播放完成后再显示下一句。
  • Preference("wait for voice", "disable") - 自动模式中不等待当前播放的语音播放完成后再显示下一句。
  • Preference("wait for voice", "toggle") - 切换是否等待语音。
  • Preference("voice sustain", "enable") - 在交互间保持语音的播放（即点击鼠标不会导致语音停止播放）
  • Preference("voice sustain", "disable") - 在交互间不保持语音的播放。
  • Preference("voice sustain", "toggle") - 在以上两种模式切换。
  • Preference("music mute", "enable") - 使music混音器静音。
  • Preference("music mute", "disable") - 取消上述设置。
  • Preference("music mute", "toggle") - 在两个模式间切换。
  • Preference("sound mute", "enable") - 使sound混音器静音。
  • Preference("sound mute", "disable") - 取消上述设置。
  • Preference("sound mute", "toggle") - 在两个模式间切换。
  • Preference("voice mute", "enable") - 使voice混音器静音。
  • Preference("voice mute", "disable") - 取消上述设置。
  • Preference("voice mute", "toggle") - 在两个模式间切换。
  • Preference("music volume", 0.5) - 设置music的音量。
  • Preference("sound volume", 0.5) - 设置sound的音量。
  • Preference("voice volume", 0.5) - 设置voice的音量。

可以作为滑条的值有：

  • Preference("text speed")
  • Preference("auto-forward time")
  • Preference("music volume")
  • Preference("sound volume")
  • Preference("voice volume")

存档文件函数¶

下面这些函数可以返回一些关于存档文件的有用信息。它们使用和文件操作一样的页码。

FileCurrentPage()¶

返回一个字符串，它是当前的页码。

FileCurrentScreenshot(empty=None, **properties)¶

返回一个可显示对象，是当前存档使用的截图，如果曾经使用 FileTakeScreenshot()进行过截图的话。

否则将显示empty。(如果empty是None，那么默认为Null()。）

FileJson(name, key=None, empty=None, missing=None, page=None)¶

检索与name存档相关联的json文件。

如果key是None，则返回整个json对象，或者empty如果存档文件不存在。

否则，该函数返回json[key]如果key在这个json对象中有定义；返回 missing如果找不到这个key，或者empty如果存档不存在。

FileLoadable(name, page=None)¶

如果存档文件可读取，返回True，否则返回False。

FileNewest(name, page=None)¶

如果存档文件是最新的存档，返回True，否则返回False。

FilePageName(auto='a', quick='q')¶

返回当前页码的名称。如果是普通的存档页，那么是一个数字；否则是 auto或quick。

FileSaveName(name, empty='', page=None)¶

返回具体的存档文件名，或者empty如果文件不存在。

FileScreenshot(name, empty=None, page=None)¶

返回存档文件相关联的截图。如果存档无法被读取，那么返回empty，除非它是None；如果真是None，那就返回一个Null()。

返回值应该是一个可显示对象。

FileSlotName(slot, slots_per_page, auto='a', quick='q', format='%s%d')¶

返回该编号的存档槽的名称。该函数假设存档槽是按顺序从1开始排列，并且页码也是从1开始的。当slot是2，slots_per_page是10的时候，其他变量是默认情况：

  • 当显示的是第一页时，返回"2"。
  • 当显示的是第二页时，返回"12"。
  • 当显示的是auto时，返回"a2"。
  • 当显示的是quick时，返回"q2"。
slot

slots_per_page

auto
自动存档页使用的前缀。
quick
快速存档页使用的前缀。
format

帮助信息条¶

Tooltip 对象是当按钮被悬停时内容发生改变的区域。

class Tooltip(default)¶

Tooltip对象可以用来将屏幕上的一块区域定义成鼠标悬停在某个区域时内容发生改变的区域。

Tooltip对象有一个value域，它的值被设置成default。当一个使用tooltip作为操作的对象被悬停时，这个value域会被修改为与这个操作相关联的值。

Action(value)¶

返回一个action对象，用来作为button的hover状态的操作。当按钮被悬停时，tooltip对象的值会被设置为。value当button失焦时，tootip的value域会变回默认值。

当使用tooltip的时候，通常的方法是使用default命令创建一个tooltip对象。在一个视屏里使用action和修改value并不一定要讲究顺序——在使用action之前使用value也是没问题的。

tooltips可以接受任意的value。在例子中我们使用text命令来显示一个字符串，但您也可以使用add命令来添加一个可显示对象。更复杂的行为也是可以的。

screen tooltip_test:

    default tt = Tooltip("No button selected.")

    frame:
        xfill True

        has vbox

        textbutton "One.":
            action Return(1)
            hovered tt.Action("The loneliest number.")

        textbutton "Two.":
            action Return(2)
            hovered tt.Action("Is what it takes.")

        textbutton "Three.":
            action Return(3)
            hovered tt.Action("A crowd.")

        text tt.value
