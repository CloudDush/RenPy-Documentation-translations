样式属性¶

样式中的样式属性决定了可显示对象的外观。在使用样式属性时，若您为其提供特定的前缀，则该样式属性将只会在特定的状态下生效，否则该样式属性会在任何状态下生效。

样式属性前缀¶

通过样式属性之前添加前缀，您可以改变一个可显示对象在被悬浮或者被选择时的外观。例如，您可以改变一个按钮的颜色来告诉玩家该按钮当前即将被选择，或者鼠标正停留在它的上方。

以下为可显示对象能够进入的五种状态：

insensitive
当用户不能与该对象交互时的状态。
idle
当该可显示对象既没有被悬浮也没有被选择时的状态。
hover
当该可显示对象被悬浮但还没有被选择时的状态。
selected_idle
当该可显示对象被选择但未被悬浮时的状态。
selected_hover
当该可显示对象被悬浮且被选择时的状态。

按钮和滑条（以及它们的变体）会在这些事件发生时更新它们自身以及所有子对象的状态。例如，如果玩家将鼠标移动到一个未被选择的按钮上，该按钮以及它的所有子对象都会进入hover状态。

样式属性前缀允许您为不同的状态设置不同的样式属性。Ren'Py中也有复合意义的前缀，这样一个前缀就能为多个状态指定样式属性。

以下为具有复合意义的前缀：

前缀 包含的状态
(no prefix) insensitive, idle, hover, selected_idle, selected_hover
insensitive_ insensitive
idle_ idle, selected_idle
hover_ hover, selected_hover
selected_ selected_idle, selected_hover
selected_idle_ selected_idle
selected_hover_ selected_hover

我们可以用一个文字按钮作为示例。文字按钮默认情况下有两种样式：button属性是指定了按钮本身的样式，而button_text指定了文字内容的样式。background样式属性可以用于设定按钮的背景，而color属性则用于设置文字颜色。例如：

init python:

 # 当此按钮为insensitive状态时背景是灰色的，为hovered状态时背景是淡蓝色的，其他时候是深蓝色的。
 style button:
     background "#006"
     insensitive_background "#444"
     hover_background "#00a"

 # 当处于selected状态时，该按钮的文字是黄色的，其他时候是绅士们最喜欢的白色的。
 style button_text:
     color = "#fff"
     selected_color = "#ff0"

样式属性值¶

样式属性的值必须是某种数据。许多值的数据类型是标准的Python数据类型，但也有一些不是。以下为这些数据类型的介绍：

position

position（位置）类型的数据描述的是一个相对于当前区域左上角的位置。（对于position类数据来说，如果当前的对象位于一个布局框中，那么当前区域指的就是这个布局框，否则指的就是整个屏幕。对于anchor类数据来说，当前区域是该对象自身所占的区域。）

Ren'Py理解position类数据的方法是由它的数据类型来决定的：

int 整形 (如 0, 1, 37, 或 42)
整数值会被理解为距离当前区域的顶部或者左边的像素个数。
float 浮点型(如 0.0, 0.5, 或 1.0)
浮点数会被理解为相对于当前区域的位置（左上角是(0,0)）。例如，0.5表示在屏幕水平方向或者垂直方向的中间，而1.0表示在屏幕的最右侧或者最下方。
renpy.absolute (如 renpy.absolute(100.25))
renpy.absolute类型的数字和整形类似（但可以不是整数），表示距离当前区域的顶部或者左边的像素个数，但只有使用亚像素级渲染的时候才有用。
displayable
任何可显示对象。
color

在Ren'Py中，Color类型的数据可以直接是一个以"#"开头，后面是3组或者4组的16进制数字的字符串，每组长度为1位或者2位。

如果是3组的话，分别表示R,G,B的值。如果是4组则表示R,G,B,A的值：:

  • "#f00" 和"#ff0000"代表不透明的红色。
  • "#0f08"和#00ff0080"代表半透明的绿色。

这些颜色的格式和HTML语言的格式一样。

Color类型的数据也可以是一个4元元组，每个元素的值是0到255之间的一个整数，分别对应RGBA的值。

  • (0, 0, 255, 255)表示不透明的蓝色。

所有样式属性¶

样式属性控制着许多可显示对象的外观。并不是所有的样式属性都适用于所有的可显示对象，所以我们已经将它们分好了组，以便讲解起来方便一些。

位置样式属性¶

该属性用于控制可显示对象相对于当前区域的位置。

xpos - position¶

相对于当前区域左侧的位置。

ypos - position¶

相对于当前区域顶部的位置。

pos - 二元组(position, position)¶

等价于将xpos设置成第一个元素的值，ypos设置成第二个元素的值。

xanchor - position¶

该可显示对象的锚点相对于该可显示对象左侧的位置。

yanchor - position¶

该可显示对象的锚点相对于该可显示对象顶部的位置。

anchor - 二元组 (position, position)¶

等价于将xanchor设置成第一个元素的值，yanchor设置成第二个元素的值。

xalign - float¶

将xpos和xanchor设置成同一个值。

yalign - float¶

将ypos和yanchor设置成同一个值。

align - 二元组(float, float)¶

等价于将xalign设置成第一个元素的值，yalign设置成第二个元素的值。

xcenter - position¶

等价于将xpos设置成这个值，同时将xanchor设置成0.5。

ycenter - position¶

等价于将ypos设置成这个值，同时将yanchor设置成0.5。

xoffset - int¶

用于指定使用xpos和xalign计算位置的时候的偏移量。

yoffset - int¶

用于指定使用ypos和yalign计算位置的时候的偏移量。

xmaximum - int¶

指定该对象的水平最大宽度，单位是像素。

ymaximum - int¶

指定该对象的垂直最大高度，单位是像素。

maximum - 二元组(int, int)¶

等价于同时设置xmaximum和ymaximum的值。

xminimum - int¶

设置可显示对象的水平最小宽度，单位是像素。只对可以改变尺寸的可显示对象有效。

yminimum - int¶

设置可显示对象的垂直最小高度，单位是像素。只对可以改变尺寸的可显示对象有效。

minimum - 元组(int, int)¶

同时设置xminimum和yminimum。

xsize - int¶

等价与同时将xmaximum和xminimum设置为这个值。可以用于设置可显示对象的宽度。

ysize - int¶

等价与同时将ymaximum和yminimum设置为这个值。可以用于设置可显示对象的高度。

xysize - 二元组(int, int)¶

等价于同时设置xsize和ysize，可以用于设置可显示对象的尺寸。

xfill - boolean¶

若为True，该可显示对象会填充所有剩余的水平区域。若为False，则其大小只会由它内部的子对象所占的区域决定。

该属性仅对可以改变尺寸的可显示对象有效。

yfill - boolean¶

若为True，该可显示对象会填充所有剩余的垂直区域。若为False，则其大小只会由它内部的子对象所占的区域决定。

该属性仅对可以改变尺寸的可显示对象有效。

area - 四元组(int, int, int, int)¶

该四元组的含义为(xpos, ypos, width, height)。这将试图把该可显示对象的左上角设置在(xpos,ypos），其尺寸为width和height。

该属性是通过将xpos, ypos, xanchor, yanchor, xmaximum, ymaximum, xminimum, yminimum, xfill, 以及 yfill设定为合适的值来完成的。

此属性不对所有的可显示对象和布局有效。

文本样式属性¶

antialias - boolean¶

默认为True，ttf字体将会开启反锯齿渲染。

black_color - color¶

当渲染基于图像的字体时，黑色会被映射成这个颜色。对ttf字体无效。

bold - boolean¶

若为True，将使用粗体渲染该字体。对于ttf字体来说，这也会同时增加字体的粗细。它也会导致文字被 config.font_replacement_map重新映射。

caret - displayable or None¶

如果不是None，必须是一个可显示对象，作为input窗口文本结尾的光标使用。如果是None，则为一个一像素宽的竖线。

color - color¶

文字颜色。如果是ttf字体，那么该字体会正常地使用这种颜色。如果是基于图像的字体，白色会映射到这种颜色（black_color可以映射黑色）。

first_indent - int¶

每个段落第一行的缩进量，单位是像素。

font - string¶

指定渲染文本使用的字体的名称。

对于ttf字体，这通常是字体文件的文件名。（比如"DejaVuSans.ttf"）。如果想从一个字体集里面载入第二个字体，该文件名可以以“编号+@”的方式开头（比如"0@font.ttc"或"1@font.ttc"）。对于基于图像的字体文件，该字符串应该是在注册这种字体时使用的名称。

size - int¶

字号大小。

italic - boolean¶

若为True，文本会使用斜体。对于ttf字体来说，这会增加字体的倾斜程度。它也会导致字体被config.font_replacement_map重新映射。

justify - boolean¶

若为True，将在单词之间插入额外的空格来保证文本的左右端是对齐的。在文本的最后一行不会使用。

kerning - float¶

调整字符之间的间距（可以是负值，将完全取消字符之间的距离）。

language - string¶

调整用于决定断行位置的语言。可选值为：

"unicode" (default)
使用Unicode断行算法，适用于大多语言。
"japanese-strict"
严格的日文断行。它严禁在小的假名和长音符号中间断行。
"japanese-normal"
普通的日文断行。它允许在小的假名和长音符号以及一些连字号中间断行
"japanese-loose"
宽松的日文断行。它允许几乎所有情况下的断行。
"korean-with-spaces"
用于带空格的韩文。这将避免在韩文单词的中间断行。
"western"
只允许在空格处断行，适用于大多西方语言。

三种日文断行方式来自CSS3文本模块.

layout - string¶

决定对齐方式。可选值有：

"tex" (默认)
使用Knuth-Plass换行算法，减少除最后一行外其他行的长度差异。
"subtitle"
使用Knuth-Plass换行算法，减少所有行的长度差异。
"greedy"
尽可能的压缩紧致。
"nowrap"
不换行。
line_leading - int¶

每行之前的空白的宽度，单位为像素。

line_overlap_split - int¶

在慢速文本模式下，当两行文字重叠时，留给上面这行文字这个像素高度的空间。若您发现上行文字显示不全，请尝试增加这个属性的值。

line_spacing - int¶

行与行之间的距离。

min_width - int¶

设置每一行的最小宽度。如果某一行比这个宽度短，它会被拉伸到这么宽，而text_align决定了填充空格的具体位置。

newline_indent - boolean¶

若为True，所有新行将使用first_indent缩进，否则使用rest_indent 缩进。

outlines - 由(int, color, int, int)组成的列表¶

这是用来描述绘制在文字后面的描边的列表。每个元组描述一个描边。

元组的元素意义为(size, color, xoffset, yoffset) tuples. Size是字体增大的程度，size越大描边越粗，单位为像素。Color是描边的颜色。xoffset 和 yoffset是描边相对于文字左上角的偏移量，单位是像素。

描边可以用于绘制文字阴影，只需将size设为0且偏移量不为0即可。

描边效果仅对ttf字体有效。

rest_indent - int¶

指定第二行之后的文本之前的缩进量。

ruby_style - style 或 None¶

如果不为None，应该是一个style对象。该样式将被应用于ruby文本。

slow_cps - int 或 True¶

如果是个数字，将以每秒该字数的速度显示文本。若为True，以Text Speed设置的速度显示文本。

slow_cps_multiplier - float¶

文本速度会乘上这个值。这可以让某些角色以超过正常速度的语速说话。

strikethrough - boolean¶

若为True，文本会带有删除线。

text_align - float¶

若文本比Text可显示对象的宽度短，将使用此方法对齐。它决定了在左边会有多少空白部分。

0.0为左对齐，0.5为居中，1.0为右对齐。

underline - boolean¶

若为True，将为文字添加下划线。

hyperlink_functions - tuple of (function, function, function)¶

这个属性是有关文本里的超链接的。

第一个函数是超链接样式函数。它被调用时传入一个参数（也就是超链接使用的参数），并返回一个style对象，比如 style.hyperlink_text。请注意style对象并不是一个字符串。

第二个函数是超链接点击函数。当用户点击超链接时被调用。如果它返回的不是None，那么该交互的返回值就是这个值。

第三个函数是超链接聚焦函数。它被调用时传入一个参数，若超链接获焦，那么就是超链接使用的参数，若超链接失焦则为None。如果它返回的不是None，那么该交互的返回值就是这个值。

vertical - boolean¶

若为True，文本会垂直方向显示。

窗口样式属性¶

窗口样式属性适用于窗口，框架和按钮对象。

background - 可显示对象或 None¶

作为窗口背景的可显示对象，通常是Frame()，因为可以配合窗口大小来调整图像大小。

若为None，那么将没有背景，但其他属性依然有效。

foreground - 可显示对象或 None¶

若不为None，该可显示对象会绘制在窗口内容的上方（前景）。

left_margin - int¶

背景上左边留出的空白边缘宽度，单位是像素。

right_margin - int¶

背景上右边留出的空白边缘宽度，单位是像素。

xmargin - int¶

等价于把上面两个属性设为同一个值。

top_margin - int¶

背景上顶部留出的空白边缘宽度，单位是像素。

bottom_margin - int¶

背景上底部留出的空白边缘宽度，单位是像素。

ymargin - int¶

等价于把上面两个属性设为同一个值。

left_padding - int¶

背景图左侧与窗口内容左侧之间的距离，单位是像素。

right_padding - int¶

背景图右侧与窗口内容右侧之间的距离，单位是像素。

xpadding - int¶

等价于把上面两个属性设为同一个值。

top_padding - int¶

背景图顶部与窗口内容顶部之间的距离，单位是像素。

bottom_padding - int¶

背景图底部与窗口内容底部之间的距离，单位是像素。

ypadding - int¶

等价于把上面两个属性设为同一个值。

size_group - string 或 None¶

如果不是None，应该是一个字符串。Ren'Py会以同样的大小来渲染这个size_group的对象。

按钮样式属性¶

child - 可显示对象或 None¶

如果不为None，该属性将会作为按钮对象的子对象。例如，该属性可以作为一个非敏感的按钮的子对象来告诉玩家这个按钮是被锁定的。

hover_sound - string¶

当按钮获焦时播放的声音。

activate_sound - string¶

当按钮被点击时播放的声音。

mouse - string¶

当按钮获焦时鼠标的样式属性。这应该是config.mouse中的一种。

focus_mask - multiple¶

决定按钮哪个部分能被获焦的蒙板。这个属性的值的类型决定了Ren'Py的处理方式。

Displayable
该可显示对象的不透明部分表示按钮可以获焦的部分。
True
使用按钮本身作为蒙板（所以按钮的不透明部分可以获焦）。.
callable
若为一个不可显示的可调用对象（带有__call__方法的函数，方法或者对象），该函数会获得两个参数（x,y）并被调用。x,y表示鼠标距离可显示对象左上角的距离。如果返回值为True，那么该可显示对象获焦。
None
若为None，整个按钮都可以获焦。

滑条样式属性¶

滑条左右带有调整按钮，按下以后可以小幅地调整滑条的位置。中间的部分则是用来决定滑条所代表的值的大小。

“滑块”指的是滑条上面那个可以被拖动的东西。

当滑条被绘制的时候，滑块的影子会先被绘制，然后是滑条的左下角和右上角，最后是滑块。

注意滑条可以滑动的方向取决于bar_vertical的值，若为True，那么是垂直的，否则是水平的。

bar_vertical - boolean¶

若为True，滑条的方向是垂直的，否则是水平的。

bar_invert - boolean¶

若为True，滑条的起点是从右/上开始的，而不是一般的左/下方。

bar_resizing - boolean¶

若为True，我们会缩放滑条的边缘。若为False，我们会先以固定尺寸渲染滑条，然后裁剪到合适的尺寸。

left_gutter - int¶

左侧调整按钮的尺寸，单位是像素。

right_gutter - int¶

右侧调整按钮的尺寸，单位是像素。

top_gutter - int¶

顶部调整按钮的尺寸，单位是像素。

bottom_gutter - int¶

底部调整按钮的尺寸，单位是像素。

left_bar - displayable¶

滑条左侧部分使用的可显示对象。

right_bar - displayable¶

滑条右侧部分使用的可显示对象。

top_bar - displayable¶

滑条上侧部分使用的可显示对象。

bottom_bar - displayable¶

滑条下侧部分使用的可显示对象。

thumb - 可显示对象 或 None¶

如果不是None，该可显示对象是中间滑块的图像。

thumb_shadow - displayable or None¶

如果不是None，该可显示对象是中间滑块的图像。（不知道和上面的有什么区别）

thumb_offset - int¶

滑块的偏移量。为了使滑条看起来连贯，请将此值设定为滑块宽度的一半。

mouse - string¶

滑条获焦时使用的鼠标样式。应该是config.mouse中的一种。

unscrollable - string 或 None¶

决定当滑条不能滑动时发生什么。有三种可能的值：

None
正常地渲染滑条。
"insensitive"
以非敏感状态渲染滑条。这将提示用户该滑条现在无效。
"hide"
不渲染滑条。会为滑条预留空间，但是不会显示任何东西。

盒子样式属性¶

这些属性是用于水平或者垂直盒状布局框的。

spacing - int¶

在盒子内的成员之间的间距，单位是像素。

first_spacing - int¶

如果不为None，则是第一个和第二个成员之间的间距。该属性优先度大于spacing属性。

box_reverse - boolean¶

如果为True，盒子内部的排列顺序会反过来。hbox对象会从右向左填充，而vbox对象会从上向下填充。默认是False。

box_wrap - boolean¶

如果为True，这些盒装布局框会在一行或者一列结束时换行。若为False（默认情况下）它们会超过行尾。

order_reverse - boolean¶

若为False，默认情况下，盒子内部的对象最先描绘的会在最下方（被后面的对象覆盖），而若为True则是最后描绘的图像会在最下方（被前面的对象覆盖）。

固定区域样式属性¶

配合Fixed布局使用。

fit_first - bool¶

若为True，该fixed对象的尺寸会缩小到它的第一个子对象的大小。
