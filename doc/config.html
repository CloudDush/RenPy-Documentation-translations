默认系统设置 Preference Defaults¶

以下是许多的系统设置的默认值。这些默认值只在第一次运行Ren'Py时，或是持续数据被删除时才生效。

These are defaults for the various preferences. The defaults only take effect the first time Ren'Py is run, or when persistent data has been cleared.

config.default_afm_time = None¶

如果不为None，本参数设置的是在自动模式下的默认翻页时间。如果不为None，本参数为每显示250个字符后需等待的秒数。0是例外，0表示无限等待，即停止自动模式。

If not None, this sets the default auto-forward-mode timeout. If not None, then this is the time in seconds we should delay when showing 250 characters. 0 is special-cased to be infinite time, disabling auto-forward mode.

config.default_afm_enable = None¶

本参数控制默认的自动模式的系统设置状态。如果为False，默认自动模式停止。如果为True，默认使用自动模式。如果为None，则忽略自动模式的系统设置项。（当自动前进速度不为0时，自动模式将生效。）

Controls the default state of the afm_enable preference. If False, auto-forward-mode is disabled by default. If True, it is enabled by default. If None, the afm_enable preferences is ignored. (Auto-forward will occur when the auto-forward speed is non-zero.)

config.default_fullscreen = None¶

本参数设置全屏模式的默认值。参数为True或False。如果为None，本参数将被忽略，允许其他代码设置默认参数值。（在options.rpy中通常设置为False。）

This sets the default value of the fullscreen preference. This shoul be True or False. If None, this is ignored, allowing other code to set the default value. (It's usually set to False in options.rpy.)

config.default_language = None¶

本参数设置语言系统设置的默认值。只在首次开始游戏时使用。

This sets the default value of the language preference. This is only used the first time the game starts.

config.default_show_empty_window = True¶

本参数决定，当没有"say"命令时，能否显示一个空窗口。如果为True，Ren'Py将响应"window show"，"window hide"和"window auto"命令。如果为False，Ren'Py将不会显示一个空窗口。

This determines if Ren'Py will show an empty window during non-say statements. If True, Ren'Py respects the "window show", "window hide", and "window auto" statements. If False, Ren'Py will not show an empty window.

config.default_text_cps = None¶

如果不为None，本参数用来设置每秒默认显示的字符数。0是例外，表示每秒显示无限多个字符。（在options.rpy文件中通常设置为0。）

If not None, this sets the default number of characters per second to show. 0 is special cased to mean an infinite number of characters per second. (It's usually set to 0 in options.rpy.)

config.default_voice_sustain = False¶

语音保持系统设置的默认值。如果值为True，在下次交互操作时，语音将保持。如果值为False，在下次交互操作时，语音将停止播放。

The default value of the voice sustain preference. If this is True, the voice will continue past the next interaction. If false, voice will stop when the next interaction begins.

config.default_wait_for_voice = True¶

语音等待的系统设置默认值。该参数决定，Ren'Py在语音播放没有结束时，是否继续自动前进。

The default value of the wait for voice preference. This determines if Ren'Py should wait for voice to finish before auto-forward takes place.

config.auto_load = None¶

如果不为None，即是Ren'Py启动时自动载入的存档文件名。该参数预计是给开发者使用，而不是面向最终玩家的。设置此参数为“1”，Ren'Py将自动载入1号档位的存档。

If not None, the name of a save file to automatically load when Ren'Py starts up. This is intended for developer use, rather than for end users. Setting this to "1" will automatically load the game in save slot 1.

config.auto_voice = None¶

此参数可以为字符串，函数，或是None。如果为None，将禁用自动语音。

This may be a string, a function, or None. If None, auto-voice is disabled.

如果为字符串，将被格式化的ID变量绑定到当前行的对话的标识符。如果给出的是存在的文件，那么文件将作为语音文件播放。

If a string, this is formatted with the id variable bound to the identifier of the current line of dialogue. If this gives an existing file, that file is played as voice audio.

如果为函数，函数将调用一个单独的数据体，当作前行的对话的标识符。函数预计会返回一个字符串。如果给出的是存在的文件，那么该文件将作为语音文件播放。

If a function, the function is called with a single argument, the identifier of the current line of dialogue. The function is expected to return a string. If this gives an existing file, that file is played as voice audio.

更多细节详见自动语音

See Automatic Voice for more details.

config.automatic_images = None¶

如果不为None，Ren'Py将自动定义图片。

If not None, this causes Ren'Py to automatically define images.

当设置不为None的时候，需要设置下列分隔符。（例如：[ ' ', '_', '/' ]）//separator翻译成了分隔符。

When not set to None, this should be set to a list of separators. (For example, [ ' ', '_', '/' ].)

Ren'Py将从磁盘和压缩文件中扫描文件列表。当找到以.png或.jpg结尾的文件，它将去掉文件扩展名，然后用分隔符拆分名字，来创建图片的名称。如果文件名由至少两部分组成，并且没有图片与已定义的名称对应。Ren'Py将定义图片关联为一个文件名。

Ren'Py will scan through the list of files on disk and in archives. When it finds a file ending with .png or .jpg, it will strip the extension, then break the name at separators, to creatge an image name. If the name consists of at least two components, and no image with that name already is defined, Ren'Py will define that image to refer to a filename.

举例说明分隔符，如果你的目录中包含：

With the example list of separators, if your game directory contains:

  • eileen_happy.png, Ren'Py将定义图片“eileen happy”。
  • lucy/mad.png, Ren'Py将定义图片“lucy mad”。
  • mary.png, Ren'Py将不做任何事情。（因为这个图片名没有两部分。）
config.automatic_images_strip = []¶

用来在定义自动图片时用来拆分前缀的字符串列表。当多个目录中包含图片时，可以用来移除目录名。

A list of strings giving prefixes that are stripped out when defining automatic images. This can be used to remove directory names, when directories contain images.

config.debug = False¶

启用除错功能（绝大多数因为丢失文件导致的错误。）在完成作品时通常应关闭选项。

Enables debugging functionality (mostly by turning some missing files into errors.) This should always be turned off in a release.

config.debug_image_cache = False¶

如果为True，只要图片缓存的内容发生变化，Ren'Py将图片缓存内容打印到标准输出。

If True, Ren'Py will print the contents of the image cache to standard output (wherever that goes) whenever the contents of the image cache change.

config.debug_sound = False¶

启用音效除错功能。禁用它将在播放声音时屏蔽报错。然而，如果声卡丢失或损坏，这些报错还是会正常生效。启用这个参数可能会妨碍Ren'Py的一些功能正常运行。在完成作品时，通常应设为False。

Enables debugging of sound functionality. This disables the supression of errors when generating sound. However, if a sound card is missing or flawed, then such errors are normal, and enabling this may prevent Ren'Py from functioning normally. This should always be False in a released game.

config.debug_text_overflow = False¶

当参数为True时，Ren'Py将记录文本溢出到text_overflow.txt文档。当Text描述的显示空间大于分配给它的显示空间会发生文本溢出。通过设置本参数为True，并且设置对话框的窗口大小的xmaximum 和 ymaximum样式属性，可以用来报告在何处会话文本超过了对话框。

When true, Ren'Py will log text overflows to text_overflow.txt. A text overflow occurs when a Text displayable renders to a size larger than that allocated to it. By setting this to True and setting the xmaximum and ymaximum style properties of the dialogue window to the window size, this can be used to report cases where the dialogue is too large for its window.

config.default_transform = ...¶

当使用show或是scene命令显示时，将从这个变换中提取属性，并用来初始化显示变换的初始值。

When a displayable is shown using the show or scene statements, the transform properties are taken from this transform and used to initialize the values of the displayable's transform.

默认的变换是center(居中)。

The default default transform is center.

config.empty_window = ...¶

当_window为True，并且视屏中没有窗口时，本参数被调用。（调用renpy.shown_window()时，不调用本参数。）它将显示一个在视屏上显示空窗口，并且不对交互动作返回响应。//这个不知道翻的对不

This is called when _window is True, and no window has been shown on the screen. (That is, no call to renpy.shown_window() has occurred.) It's expected to show an empty window on the screen, and return without causing an interaction.

默认显示一个没有交互的空白行。

The default implementation of this uses the narrator character to display a blank line without interacting.

这个选项决定了，在锁定回滚的时候，如何创建菜单或是图片映射的行为。默认值为False，意味着，菜单之前的选项仍然是可以点选的。如果设置为True，选项将会记录，没有选项可被选择。用户可以通过点击回滚缓冲来向前发展。（晕了……，相当不确定。意思理解不明确，怎么翻都不顺。）

This option determines how the built in menus or imagemaps behave during fixed rollback. The default value is False, which means that menu only the previously selected option remains clickable. If set to True, the selected option is marked but no options are clickable. The user can progress forward through the rollback buffer by clicking.

config.font_replacement_map = { }¶

这是一个（字体，粗体，斜体）到（字体，粗体，斜体）的映射，使用粗体或斜体的字体来替代原字体。例如，如果你想要所有使用斜体版本的“Vera.ttf”字体用“VeraIt.ttf”字体来替换，你可以这样写：

This is a map from (font, bold, italics) to (font, bold, italics), used to replace a font with one that's specialized as having bold and/or italics. For example, if you wanted to have everything using an italic version of "Vera.ttf" use "VeraIt.ttf" instead, you could write:

init python:
    config.font_replacement_map["Vera.ttf", False, True] = ("VeraIt.ttf", False, False).

请注意，此映射只允许在字体的特殊变体上。既然这样，需要一个粗体斜体版本的vera将得到一个粗体和斜体的版本，而不是一个粗体版本的斜体vera。

Please note that these mappings only apply to specific variants of a font. In this case, requests for a bold italic version of vera will get a bold italic version of vera, rather than a bold version of the italic vera.

config.framerate = 100¶

如果不为None，这是Ren'Py每秒显示帧率的上限数值。这只是关系软件转换。GL转换将另使用垂直行同步方式替代。

If not None, this is the upper limit on the number of frames Ren'Py will attempt to display per second. This is only respected by the software renderer. The GL renderer will synchronize to vertical blank instead.

config.image_cache_size = 8¶

本参数在多视屏的情况下，用来设置图片缓存区的大小。随着视屏尺寸的增多，图片缓存将占用更大的空间。

如果设置过大，将会浪费内存。如果设置过小，图片可能会反复载入，影响执行。

If set too large, this can waste memory. If set too small, images can be repeatedly loaded, hurting performance.

config.key_repeat = (.3, .03)¶

控制键盘连击的速率。当启用了键盘连击，参数为一个元组。元组的第一项为第一次连击的延时，第二项为之后各连击之间的延时。两条都是以秒为单位。如果为None，键盘将禁用连击。

Controls the rate of keyboard repeat. When key repeat is enabled, this should be a tuple. The first item in the tuple is the delay before the first repeat, and the second item is the delay between repeats. Both are in seconds. If None, keyboard repeat is disabled.

config.mouse = None¶

本参数控制用户自定义的鼠标指针。如果为None，将使用系统默认的黑白色的鼠标指针图标。

This variable controls the use of user-defined mouse cursors. If None, the system mouse is used, which is usually a black-and-white mouse cursor.

否则，此处应是一个字典，给出多种鼠标类型的鼠标动画。图标的默认库包括：“default”,“say”,“with”,“menu”,“prompt”,“imagemap”,“pause”,“mainmenu”和“gamemenu”。default图标经常出现，因为当没有其他明确状态时，都使用default。

Otherwise, this should be a dictionary giving the mouse animations for various mouse types. Keys used by the default library include "default", "say", "with", "menu", "prompt", "imagemap", "pause", "mainmenu", and "gamemenu". The "default" key should always be present, as it is used when a more specific key is absent.

字典中的每个值都应该为(image, xoffset, offset)格式的元组。

Each value in the dictionary should be a list of (image,xoffset, offset) tuples, representing frames.

image
鼠标的指针图标。
xoffset
焦点像素与指针图标左侧的偏差。
yoffset
焦点像素与指针图标上侧的偏差。

鼠标指针结构重放频率为20hz，而且当播放全部后，会从头重放。

image
The mouse cursor image.
xoffset
The offset of the hotspot pixel from the left side of the cursor.
yoffset
The offset of the hotspot pixel from the top of the cursor.

The frames are played back at 20hz, and the animation loops after all frames have been shown.

A list of archive files that will be searched for images and other data. The entries in this should consist of strings giving the base names of archive files, without the .rpa extension.

归档文件按照列表中的顺序进行寻找。文件将从首次找到的归档中提取。

The archives are searched in the order they are found in this list. A file is taken from the first archive it is found in.

在启动时，Ren'Py将按照ascii码的逆序来排列在目录下找到的所有归档文件，并写入到本参数。例如：Ren'Py找到了文件“data.rpa”，“patch01.rpa”，“patch02.rpa”，那么本变量将被填写成['patch02', 'patch01', 'data']。

At startup, Ren'Py will automatically populate this variable with the names of all archives found in the game directory, sorted in reverse ascii order. For example, if Ren'Py finds the files data.rpa, patch01.rpa, and patch02.rpa, this variable will be populated with ['patch02', 'patch01', 'data'].

config.auto_choice_delay = None¶

如果设置不为None，本参数是以秒为单位的时间。在游戏出现内部菜单暂停时，经过设置的时间之后，Ren'Py会随机选择选项继续游戏。我们预计所有完成的游戏都会设置为None，不过如果在此设置一个值的话，可以自动演示游戏，而不需要很多的互动。

If not None, this variable gives a number of seconds that Ren'Py will pause at an in-game menu before picking a random choice from that menu. We'd expect this variable to always be set to None in released games, but setting it to a number will allow for automated demonstrations of games without much human interaction.

config.autosave_frequency = 200¶

粗略的，在进行多少次交互后进行一次自动保存。如果要关闭自动保存，请设置config.has_autosave参数为False，而不要修改这个参数。

Roughly, the number of interactions that will occur before an autosave occurs. To disable autosaving, set config.has_autosave to False, don't change this variable.

The default value of the callback parameter of Character.

config.fade_music = 0.0¶

在新音乐开始播放前，旧音轨淡出的秒数。通常设置很短，使错误的音乐不会播放太久。

This is the amount of time in seconds to spend fading the old track out before a new music track starts. This should probably be fairly short, so the wrong music doesn't play for too long.
